# photo-tools

Auto-tag photos and manage image metadata using RAM++, CLIP landmark lookup, PaddleOCR, GPS reverse geocoding, and EXIF analysis. Writes IPTC/XMP keywords with DigiKam hierarchical tag support.

## Features

- **RAM++ tagging** — Recognize Anything Plus Plus for multi-label image content recognition (animals, scenes, objects, food, etc.)
- **Landmark lookup** — CLIP embeddings + GPS radius filtering against a FAISS-indexed Wikidata landmark database
- **OCR text detection** — PaddleOCR extracts visible text and writes it as `text/` tags with IPTC ImageRegion metadata
- **GPS geocoding** — Reverse geocode GPS coordinates into `country/`, `city/`, `region/`, `neighborhood/` tags via Nominatim
- **EXIF metadata** — Extract date tags (`year/`, `month/`, `day/`), flash, screenshot detection, weekend/weekday
- **GPS timeline inference** — Images without GPS inherit coordinates from nearby photos taken within 30 minutes
- **Similar image detection** — Find visually similar images using CLIP embeddings with interactive dedup session
- **Tag management** — List, search, delete, rename tags across image collections
- **DigiKam integration** — Clean up autotag-generated tags from DigiKam's SQLite database
- **Landmark database builder** — Scrape Wikidata for notable landmarks and build a CLIP embedding index
- **Watch mode** — Monitor a directory and auto-tag new images as they appear

## Quickstart

```bash
# Install
uv sync

# Auto-tag a directory of photos
photo-tools tag /path/to/photos -r

# Preview tags without writing (dry run)
photo-tools tag /path/to/photos -r --dry-run

# Run only specific pipelines
photo-tools tag /path/to/photos --ram          # RAM++ content tags only
photo-tools tag /path/to/photos --ocr          # OCR text detection only
photo-tools tag /path/to/photos --landmarks    # landmark lookup only

# Re-tag (clears previous autogenerated tags first)
photo-tools tag /path/to/photos -r --force

# Watch for new images
photo-tools tag /path/to/photos -r --watch

# Find similar images
photo-tools find-similar /path/to/photos -r

# Tag management
photo-tools tags list /path/to/photos -r
photo-tools tags search /path/to/photos "animal/cat"
photo-tools tags delete /path/to/photos "old-tag"
photo-tools tags rename /path/to/photos "old-tag" "new-tag"

# Build landmark database from Wikidata
photo-tools build-landmarks --test            # small test set (Rome + Bologna)
photo-tools build-landmarks -l 5000           # full build, 5000 landmarks

# Clean DigiKam database
photo-tools drop-digikam-tags /path/to/digikam4.db            # dry run
photo-tools drop-digikam-tags /path/to/digikam4.db --execute   # actually delete

# Verbose logging
photo-tools -v tag /path/to/photos

# Custom config overlay
photo-tools --config my_config.yaml tag /path/to/photos
```

## Architecture

```
photo_tools/
  cli.py               Single CLI entry point, all subcommands
  config.py            YAML config loader (defaults + user overlay)
  default_config.yaml  All configurable thresholds and parameters
  logging_setup.py     Centralized logging configuration
  constants.py         Shared constants (tag prefixes, extensions, patterns)
  helpers.py           Exiftool ops, file discovery, image prep, embedding cache
  taxonomy.py          Tag category definitions and per-category limits
  autotag.py           Core tagging engine (RAM++, GPS, OCR, EXIF, landmarks)
  clip_tagger.py       CLIP image embedder (ViT-B-32)
  ram_tagger.py        RAM++ image tagger
  landmarks.py         FAISS-backed landmark lookup
  duplicates.py        Similar image detection + tag management
  build_landmarks.py   Wikidata landmark scraper
  drop_digikam_tags.py DigiKam tag cleanup
  data/
    ram_tag_mapping.json  RAM++ tag → taxonomy mapping (14k entries)
```

**Dependency flow** (no circular imports):

```
constants, config, logging_setup, taxonomy   (leaf modules)
         ↓
       helpers                               (exiftool, file I/O, image prep)
         ↓
  clip_tagger, ram_tagger, landmarks         (ML models)
         ↓
  autotag, duplicates, build_landmarks       (pipelines)
         ↓
       cli                                   (entry point)
```

All magic numbers and thresholds live in `default_config.yaml` and can be overridden with `--config path/to/overrides.yaml`.
